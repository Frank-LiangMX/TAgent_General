# docs/reports

> 周期性或一次性产出的**报告型文档**。不同于 `plans/`（设计规划）和 `decisions/`（架构决策），`reports/` 记录**已经发生过的事实**和**事后复盘**。

## 命名规范

```
reports/YYYY-MM-DD-<topic>.md
```

例：

- `2026-06-24-doc-cleanup-report.md` — 文档整理报告
- `2026-06-XX-ta-mcp-rollout-retro.md` — TA MCP 上线后复盘

## 与其他目录的关系

| 目录 | 性质 | 时态 |
|---|---|---|
| `plans/` | 设计、规划 | 未来 |
| `decisions/` | 架构决策 | 决策时点 |
| `process/` | 流程规范 | 持续 |
| `reports/` | 复盘、报告 | 已发生 |
| `archive/` | 归档 | 历史 |
